Lesson 03 · Memory & RAG

Giving agents a memory

An LLM has no recall between calls. To hold a real conversation — or work with your private documents — we hand it a notebook. We'll compare two ways of taking notes and then see how agents do their own research with Agentic RAG.

PART 01

Why memory matters

Every LLM call starts amnesiac

If you told the model your name a minute ago, it doesn't remember. Each API call is a clean slate. The trick: before the next call, we paste the past conversation back in. That bundle of past messages is the memory.

▶ See why memory is needed for real conversations
PART 02

Buffer Memory vs Summary Memory

Keep every word, or keep the gist

ConversationBufferMemory stores every message, word for word — perfect recall but the prompt balloons. ConversationSummaryMemory asks the LLM to compress old turns into a short summary — cheap and compact, but loses fine detail.

▶ Compare: keeping everything vs keeping the gist
BUFFER MEMORY

Perfect recall, growing cost

Best for: short chats, debugging

  • Stores every message word-for-word
  • Zero info loss
  • Prompt grows with each turn → token cost rises
  • Hits context limit on long sessions
SUMMARY MEMORY

Compact gist, constant cost

Best for: long chats, support bots

  • LLM compresses old turns into a summary
  • Stays small no matter how long the chat is
  • Loses exact wording (and sometimes nuance)
  • Each summary call adds a small LLM cost
from langchain.memory import ConversationBufferMemory, ConversationSummaryMemory buffer = ConversationBufferMemory() # keep everything summary = ConversationSummaryMemory(llm=llm) # keep a gist
PART 03

Agentic RAG — letting the agent decide

From "always look up" to "look up if needed"

Plain RAG (Retrieval-Augmented Generation) is a chain: every question goes to the vector store first, then to the LLM. Agentic RAG turns the retriever into a tool — the agent decides when to query it, what to query, and whether one search is enough. Smarter, but more expensive.

▶ See plain RAG vs Agentic RAG side by side
# Agentic RAG = wrap the retriever as a normal tool from langchain.tools.retriever import create_retriever_tool retriever_tool = create_retriever_tool( vectorstore.as_retriever(), name="company_docs", description="Look up internal company policies and product docs.", ) tools = [retriever_tool, search] agent = create_react_agent(llm, tools, prompt)
Plain RAG: 1 retrieval + 1 LLM call · fast · cheap Agentic RAG: 0–N retrievals + multiple LLM calls · smarter · pricier Use agentic when: queries vary, need follow-ups, multiple sources
