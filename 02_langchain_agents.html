Lesson 02 · LangChain Agents

Building agents with LangChain

LangChain hands you a ready-made cockpit: the Agent Executor drives the ReAct loop while tools do the actual work. Let's wire one up step by step — a search tool, a Python tool, and one of your own.

PART 01

The Agent Executor — your cockpit

Manages the loop so you don't have to

In LangChain, you don't hand-roll the ReAct loop. The Agent Executor is a wrapper that holds three things: an LLM, a list of tools, and a prompt. It calls the LLM, parses the tool request, runs the tool, feeds the result back — and repeats until the LLM says "done".

▶ See the three pieces an Executor wires together
# the executor binds it all together from langchain.agents import AgentExecutor, create_react_agent agent = create_react_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) executor.invoke({"input": "weather in Kolkata today?"})
PART 02

Tool #1 — DuckDuckGo search

Give the agent the internet, for free

DuckDuckGo's search is one of the friendliest tools to start with — no API key, no signup. You import it, hand it to the agent, and the LLM gets a new ability: "if I don't know, I can search."

▶ Watch the agent call DuckDuckGo and read results
from langchain_community.tools import DuckDuckGoSearchRun search = DuckDuckGoSearchRun() tools = [search] # the agent can now reach the web
PART 03

Tool #2 — Python REPL

Hand the agent a calculator (a very smart one)

LLMs are famously shaky at arithmetic and date math. The fix: don't ask them to do it — ask them to write code that does it. A Python REPL tool runs whatever code the agent generates and returns the printed output as an observation.

▶ Watch the agent write code, run it, and read the result
from langchain_experimental.tools import PythonREPLTool python_repl = PythonREPLTool() tools = [search, python_repl] # now agent can both search AND compute
⚠ warning: REPL runs arbitrary code — never expose to untrusted users without sandboxing
PART 04

Tool #3 — Write your own

Any Python function + a good description = a tool

The real superpower of LangChain agents is that any function you write can become a tool. Three things matter: a clear name, a precise docstring (the LLM reads it to decide when to use the tool), and well-typed arguments. The @tool decorator handles the rest.

▶ Turn any Python function into a tool the agent can use
from langchain.tools import tool @tool def get_stock_price(symbol: str) -> str: """Return the latest live price for a stock ticker, e.g. 'AAPL'.""" return yfinance.Ticker(symbol).info["regularMarketPrice"] tools = [search, python_repl, get_stock_price] executor = AgentExecutor(agent=create_react_agent(llm, tools, prompt), tools=tools)
📛

Clear name

The function name is what the LLM sees first. Pick verbs: get_, search_, send_.

get_stock_price ✓ · stk ✗
📝

Precise docstring

The LLM reads this to decide when to call your tool. Say what it does and when to use it.

"Return live price for a stock ticker"
🔠

Typed arguments

Annotate types — the LLM uses them to fill in the right kind of value.

symbol: str → "AAPL"
