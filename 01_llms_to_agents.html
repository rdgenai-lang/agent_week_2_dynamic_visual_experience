Lesson 01 · Agents 101

From LLMs to Agents

A language model can talk. An agent can think, choose tools, observe results, and try again. Let's see exactly what changes when we give a model hands and a memory.

PART 01

The plain LLM

Input → Output. Nothing else.

An LLM by itself is a brilliant talker locked in a room. You slide a question under the door, it slides an answer back. It can't check the news, run code, or remember yesterday. For anything happening right now, it has to guess — or admit defeat.

▶ Press play to see the LLM hit its wall
knows: general facts from training can't: check live data can't: run code or call APIs can't: remember past chats
PART 02

The ReAct loop — how an agent thinks

Reason · Act · Observe · repeat

An agent thinks out loud, picks a tool, uses it, and reads what came back — then decides if it's done or needs another turn. This little circle is the heart of every agent you'll meet. The name says it all: Reason + Act = ReAct.

▶ Walk through one full ReAct loop
Thought: "I need live data — let me search." Action: call search("weather Kolkata") Observation: "Sunny, 24°C" Decide: done → answer the user
PART 03

Chain vs Agent

A railway track vs a taxi driver

A chain is a fixed pipeline — A → B → C, every single time, no matter what. An agent picks the next step itself based on what it just saw, so the path can branch, loop, or skip steps. Same building blocks, very different freedom.

▶ See how agents branch where chains can't
CHAIN

Predictable & cheap

  • Steps decided by you, in advance
  • Same path every run — easy to test
  • Great for: summarise, translate, format
  • Breaks when the task needs a decision
AGENT

Flexible & powerful

  • Steps decided by the model, at runtime
  • Can branch, retry, loop, skip
  • Great for: research, debugging, multi-step jobs
  • Costs more tokens & can wander — needs guardrails
PART 04

Why complex tasks need agents

Three things a chain just can't do

Imagine asking: "Find the cheapest flight from Kolkata to Tokyo next Friday, check if my passport is still valid, and put it on my calendar." No single LLM call can do this. The path branches, depends on real data, and may need to retry. That's agent territory.

▶ Watch one request fan out into many tool calls
🌿

Branching paths

Real tasks have "if this, then that" decisions a fixed chain can't take.

🛠

Live tools

Search, code, databases, calendars — the world keeps changing, agents reach for it.

🔁

Recover from errors

Tool failed? Empty result? An agent can retry, ask differently, or try a different tool.

Rule of thumb: if you'd write "depends on…" in the instructions, you want an agent.
