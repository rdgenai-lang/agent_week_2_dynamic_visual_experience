Lesson 04 · LangGraph

Welcome to LangGraph

LangChain agents are great until your workflow grows past "search → think → answer." LangGraph models an agent as an explicit graph of nodes sharing a single state — perfect for branching logic, retries, and human-in-the-loop pauses.

PART 01

Stateful agents — one shared notebook

Every node reads and writes the same state

In LangGraph, your agent is a graph: nodes are pieces of work (an LLM call, a tool, a parser), and they all share one Python dict called the state. A node reads what's there, does something, and writes back. The next node sees the update. Simple — but it changes everything.

▶ Watch nodes share one piece of state
from langgraph.graph import StateGraph from typing import TypedDict class AgentState(TypedDict): query: str results: list answer: str graph = StateGraph(AgentState) graph.add_node("planner", planner_fn) graph.add_node("searcher", searcher_fn) graph.add_node("writer", writer_fn)
PART 02

Conditional edges — letting state choose the next step

A function decides which arrow to follow

In a regular graph, edges are fixed: node A always goes to B. In LangGraph you can add a conditional edge — a tiny Python function that reads the current state and returns the name of the next node. That's how an agent decides "do I have enough info, or should I search again?"

▶ See state pick the path at runtime
def should_continue(state: AgentState) -> str: if state["messages"][-1].tool_calls: return "tool_node" # needs a tool, go run one return "END" # done, exit the graph graph.add_conditional_edges("agent", should_continue)
Branch: "if cart_total > 1000 → manager_node else → auto_approve" Retry: "if error → retry_node (up to 3 times)" Loop: "while not satisfied → keep thinking"
PART 03

Human-in-the-Loop checkpoints

Pause the graph, wait for a person, resume

For high-stakes actions — sending money, deleting data, posting publicly — you don't want the agent to act alone. LangGraph's checkpointer can pause execution before sensitive nodes, save full state to disk, and wait. A human reviews, approves or edits, and the graph picks up exactly where it left off.

▶ See an agent pause for human approval
from langgraph.checkpoint.memory import MemorySaver checkpointer = MemorySaver() app = graph.compile(checkpointer=checkpointer, interrupt_before=["send_email"]) # run until the checkpoint, then pause app.invoke(state, config={"configurable": {"thread_id": "user-42"}}) # later — human approves, resume from saved state app.invoke(None, config={"configurable": {"thread_id": "user-42"}})
⏸

Pauses cleanly

Stop before any node — the agent never half-executes a critical step.

💾

Durable state

State is serialized to disk (or DB), so pauses can last seconds or days.

✏️

Editable

Humans can tweak state mid-flow before resuming — not just approve/reject.
